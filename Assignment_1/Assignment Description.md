# Gemini API + Manim Animation Assignment

## Overview

This project demonstrates the use of Google's Gemini API to automatically generate Manim animations for mathematical concepts. The generated code was executed using Manim Community Edition, and the resulting animations were critically evaluated to identify limitations in correctness, visualization quality, and maintainability.

## Objectives

1. Generate Manim animation code using the Gemini API.
2. Render and evaluate the generated animations.
3. Critically analyze the strengths and weaknesses of AI-generated code.
4. Document the workflow and findings in a GitHub repository.

---

## Task 1: Pythagorean Theorem Visualization

### Description

A Manim animation was generated to visually demonstrate the Pythagorean Theorem:

[
a^2 + b^2 = c^2
]

The animation includes:

* A right triangle
* Labels for sides (a), (b), and (c)
* Squares constructed on each side
* Color-coded regions
* Display of the algebraic identity

### Files

* `generate_pythagoras.py` – Gemini API prompt and generation script
* `pythagoras.py` – Raw Manim code generated by Gemini
* `pythagoras_output.mp4` – Rendered animation
* `pythagoras_screenshot.png` – Screenshot of the final output
* `critique_pythagoras.md` – Critical analysis

### Critical Findings

The generated animation successfully visualized the theorem but exhibited several limitations, including geometry handling, layout choices, labeling clarity, and animation pacing. These issues were documented and analyzed in detail.

## Task 2: Fourier Series Decomposition

### Description

A Manim animation was generated to illustrate how a square wave can be approximated using the Fourier Series by progressively adding odd harmonics.

The animation includes:

* Coordinate axes
* Individual harmonic sine waves
* Distinct colors for each harmonic
* Progressive cumulative summation
* Approximation of a square wave

### Files

* `generate_fourier.py` – Gemini API prompt and generation script
* `fourier_series.py` – Raw Manim code generated by Gemini
* `fourier_output.mp4` – Rendered animation
* `fourier_screenshot.png` – Screenshot of the final output
* `critique_fourier.md` – Critical analysis

### Critical Findings

Several shortcomings were observed in the generated code, including the use of deprecated Manim APIs, incorrect object manipulation, axis-label formatting issues, label overlap, and limited visual emphasis on the final Fourier approximation.

## Conclusion

This assignment demonstrates both the capabilities and limitations of large language models for code generation. Gemini successfully generated functional Manim code for complex mathematical visualizations; however, the generated outputs required inspection, debugging, and refinement. The critical analysis highlights the importance of human review when using AI-generated code for technical and educational applications.
